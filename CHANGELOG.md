# Changelog

此文件記錄 **CSI Portal** 專案的所有重大變更。
版本號遵循 [語義化版本 (SemVer)](https://semver.org/lang/zh-TW/) 規範：
* **Major (X)**：重大架構改動（如更換核心框架）。
* **Minor (Y)**：新增功能。
* **Patch (Z)**：Bug 修復或效能優化。

---

## [1.8.1] - 2026-03-13
### Added (新增)
* **混合雙引擎與多模態影音直讀**：
    * 導入 Google File API：全面升級 AI 核心架構，新增支援 .pdf 與 .mp4 / .mov 影音格式。AI 現在能具備「視覺」與「聽覺」，直接讀取 PDF 內的複雜圖表、數學公式，甚至能觀看教學影片、聽取語音並解讀簡報畫面，產出精準考題。
    * Word 純文字引擎解放：針對 .docx 檔案保留高穩定性的純文字萃取模式，並將擷取字數上限從 3 萬字大幅解放至 50 萬字，足以應付數百頁的長篇講義。

* **歷屆考題專屬模式**：
    * 刷題神器機制：於上傳教材表單新增「這是歷屆考題」核取方塊（綁定 Material 模型之 is_exam_paper 欄位）。
    * 智慧任務切換：勾選後系統將動態切換 AI Prompt，省略重點摘要生成，強制要求 AI 專注於提取文件中的考題，並針對無答案的考卷自動運算正解，大幅提升備考效率。

* **AI 錯題詳解與追蹤系統**：
    * 自動生成詳解：擴充資料庫 QuizMistake 模型新增 explanation 欄位。AI 出題時將同步產出「為何正確與為何錯誤」的深度解析。
    * 詳解視覺化介面：測驗結果報表新增專屬的「💡 AI 解析」燈泡區塊，幫助使用者真正理解錯題觀念，而不只是死背答案。
    * 原題號追蹤：錯題標題自動附加原始考卷的題號 (例如：Q3.)，方便使用者快速對照原始教材上下文。

### Changed (變更)
* **系統強健性與底層防呆機制**：
    * 中文檔名編碼防護：為徹底解決 Google GenAI SDK 底層的 ascii 中文路徑編碼報錯 Bug，實作了本機 tempfile 暫存機制。系統會在背景自動生成純英文替身檔案進行上傳，確保各類語言檔名皆能暢通無阻。
    * 雙重閱後即焚：強化隱私與空間管理，檔案交由 AI 解析完畢後，確保 Google 遠端伺服器與本機端的純英文暫存檔皆被徹底刪除，達成零殘留。
    * 攔截器白名單：修正視圖層的空檔案攔截邏輯，將影音格式加入白名單，避免無文字內容的 MP4 遭系統誤擋。

* **使用者體驗 (UI/UX) 與介面優化**：
    * 會員專區收納：重構頂部導覽列，將點數儲值、點數商城、投資儀表板與 AI 教材大腦四大功能，優雅地收納進全新的「會員專區」下拉選單中，並加入動態 Active 狀態與 fa-fw 圖示對齊，維持畫面的清爽專業。
    * 樣板渲染修復：引入 Django {% resetcycle %} 標籤，解決選擇題字母 (A, B, C, D) 跨題連續輪替導致出現 E、F 選項的顯示異常；並優化錯題卡片的 Flexbox CSS，使用 flex-grow-1 確保長文選項不溢出且左右兩欄完美等高。

## [1.8.0] - 2026-03-11
### Added (新增)
* **AI 教材大腦與無限擴充題庫**：
    * 智慧文檔解析與生成：全新建立 study_brain 模組，支援上傳 PDF 與 Word 格式教材，並自動萃取文本交由 Google Gemini API產生條理分明的 Markdown 重點摘要與結構化 JSON 選擇題。
    * 題庫無限擴充機制：實作智慧疊加演算法，對同一份教材重複點擊「訓練 AI」時，系統會自動省略摘要生成以節省 Token，並將歷史題目作為黑名單餵給 AI，強制產出全新且具深度的「情境應用題」，動態擴充題庫。

* **沉浸式學習室與追蹤系統**：
    * 互動測驗與結果報表：打造專屬學習室，實作前端選項點擊與作答邏輯，提交後自動結算成績，導向專屬「測驗結果報表」，以視覺化卡片呈現正確率、錯誤率，並提供左紅右綠的錯題比對介面。
    * 個人歷史紀錄儀表板：新增「My Quiz History」頁面，完整列表呈現使用者過去的所有測驗時間、得分與教材來源，支援一鍵點擊重溫歷史錯題，打造完整的學習閉環。

* **全站教材共享與收藏機制**：
    * 社群學習知識庫：打破單機限制，上傳的教材預設為全站公開。儀表板右側新增「我的教材庫」與「探索公開教材」雙頁籤切換功能。
    * 非破壞性收藏邏輯：導入 ManyToManyField 實作 saved_by 收藏機制，使用者可自由將公開教材加入或移出個人圖書館，確保檔案實體與 AI 訓練結果可供全站共享而不被誤刪。

### Changed (變更)
* **系統強健性與 API 防護機制**：
    * AI 伺服器忙碌攔截：優化 utils.py 中的 API 呼叫邏輯，精準捕捉 503 Service Unavailable 與 429 錯誤。捨棄背景無效空轉，改為「快速失敗」並回傳清晰的錯誤訊息，配合 Django messages 與前端 JavaScript alert() 實作即時跳窗提醒。
    * 正向查詢優化：全面修正 Django ORM 查詢邏輯，捨棄依賴 related_name 的動態反向查詢，改採明確的正向過濾，徹底消除靜態型別檢查器的誤判紅線。

* **使用者體驗 (UI/UX) 與介面渲染優化**：
    * Markdown 即時渲染：前端引入 marked.js 引擎，將 AI 產出的純文字 Markdown 標記即時轉換為具備層次感的高質感 HTML 排版（粗體藍字標題、圓點清單）。
    * 懸浮置頂進度條：於測驗頁面導入 position: sticky 與毛玻璃 CSS 特效，讓「已作答/未作答」進度統計條在使用者向下捲動長篇題目時，始終懸浮於畫面頂端。
    * 防呆動畫與原生預覽：實作表單送出攔截機制，點擊「訓練 AI」後按鈕即刻禁用並顯示 Spinner 旋轉動畫，防止重複扣除配額，新增檔案線上預覽端點，支援瀏覽器內嵌分頁直接閱讀 PDF 原檔。

## [1.7.0] - 2026-03-09
### Added (新增)
* **投資儀表板與數據視覺化**：
    * 已實現損益與股息追蹤：於 Transaction 模型新增 realized_pnl 欄位，支援賣出價差與現金股息之自動結算，並於儀表板新增「落袋為安」總資產數據卡片。
    * Chart.js 互動圖表整合：實作個人資產配置甜甜圈圖，以及市場數據掃描功能（漲跌幅 Top 10、千金/銅板股 Top 10 柱狀圖）。
    * 即時個股走勢圖 API：新增動態抓取歷史股價之 API 端點，支援前端透過 AJAX 點擊持股代碼或使用搜尋框輸入，即時彈出近 6 個月平滑折線圖。

* **系統自動化排程**：
    * 背景排程器導入：整合 django-apscheduler，實作 BackgroundScheduler 處理每日收盤後 (14:30) 之台股報價自動更新。
    * 平滑中斷機制：主執行緒實作休眠迴圈與 KeyboardInterrupt 捕捉機制，確保終端機可以透過 Ctrl+C 安全關閉排程器而不會假死。

### Changed (變更)
* **報價爬蟲效能與強健性優化**：
    * 極致批量寫入：將千檔股票之迴圈儲存邏輯，全面改寫為記憶體暫存與 bulk_create / bulk_update 批量操作，並配合 select_related 解決 N+1 Query 問題，使資料庫 I/O 時間從數分鐘大幅縮短至 1 秒內。
    * 例外防呆機制：修復 Yahoo Finance 回傳空值 (NaN) 導致的 decimal.ConversionSyntax 錯誤，並加入自動淘汰機制，遇下市或無效代碼時自動標記為 is_active=False，保持資料庫純淨。

* **使用者體驗 (UI/UX) 與介面調整**：
    * 圖表收納機制：為避免頁面過長，運用 Bootstrap Collapse 元件將大盤統計圖表預設隱藏，加入專屬工具列支援一鍵展開與個股快速搜尋。
    * 防呆與動畫：優化 AI 理財規劃表單，送出後按鈕自動禁用並顯示 loading 旋轉動畫，防止使用者連點造成重複發送 API 請求。

## [1.6.0] - 2026-03-04
### Added (新增)
* **金流與點數系統**：
    * 綠界科技 (ECPay) 整合：實作線上儲值功能，支援動態 ReturnURL 抓取與 CheckMacValue 簽章驗證機制。
    * 點數商城：開發商品兌換功能，支援「儲值點數」與「紅利點數」混合折抵邏輯，並具備庫存即時扣除機制。
    * 錢包儀表板：於個人檔案整合點數資產總覽（儲值/紅利）與分頁式交易歷史紀錄（每頁 10 筆）。

* **互動體驗 (UI/UX)**：
    * 操作確認機制：商品兌換新增 Bootstrap Modal 二次確認視窗，防止使用者誤觸扣點。
    * 動態回饋彈窗：實作兌換成功後的滿版綠色成功視窗，支援隨意點擊關閉，提升操作儀式感。

### Changed (變更)
* **安定申報比對程式**：
    * 複合標題合併邏輯：支援 Excel 跨行標題自動合併（處理 start_row-3 與 start_row-4），並具備空值自動遞補與底線連接功能。
    * 科學符號 (Scientific Notation) 精度處理：引入 decimal.Decimal 進行高精度比對，並設定相對誤差門檻值 ($1 \times 10^{-5}$)，解決 Excel 長數字四捨五入導致的比對失效問題。
    * 資料正規化：優化日期格式（連字號轉純數字）與字串清洗邏輯，確保資料庫與申報檔欄位一致性。

* **國際化與錯誤修正**：
    * 多國語言補充：新增個人檔案錢包區、交易紀錄表與商城操作相關之繁體中文與英文翻譯。
    * SDK 兼容性修正：調整 ecpay_payment_sdk 呼叫邏輯，改用 generate_check_value 確保本機端與測試網域連線正常。
